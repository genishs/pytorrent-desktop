# UX Spec — pytorrent-desktop (MVP v0.1)

Owner: product-planner. Source of truth for **what** and **why**: [`SCOPE.md`](SCOPE.md).
This document concretizes SCOPE's 11 MVP acceptance criteria into screen-level layout,
states, copy, and interactions so design-publisher can build the visuals and developer
can wire the logic without further interpretation.

**Legend used throughout:**
- **MUST** — required to satisfy one of SCOPE's 11 MVP acceptance criteria. Not
  optional; cutting it fails MVP.
- **SHOULD** — reasonable UX completeness the planner added (e.g. tooltips, an
  "open folder" menu item). Safe to defer post-MVP if dev-lead needs to cut scope;
  does not block the 11 criteria.
- **OPEN** — a decision or engine change this spec assumes, flagged back to
  product-lead / dev-lead for confirmation before or during build.

## 0. Data model this spec assumes

The UI has no state of its own — it renders `TorrentEngine.snapshot()`
(`src/pytorrent_desktop/core/engine.py`) once per second. Current `TorrentStatus`
fields: `info_hash, name, total_bytes, progress (0.0-1.0), download_rate,
upload_rate, num_peers, state (str), is_paused (bool)`.

**OPEN-1 (to dev-lead):** this spec's state machine (§5) needs two fields the
current dataclass doesn't expose yet:
- `error_message: str | None` — libtorrent's `torrent_status.error` /
  `errc`. Without it the UI can only show "an error occurred," not why.
- `queue_position: int` (libtorrent already exposes this on `torrent_status`) —
  needed to distinguish "queued, waiting its turn" from "active" when the
  sequential queue is on. Without it, a queued torrent and a stalled active
  torrent are visually indistinguishable.

Both are additive, non-breaking changes to `TorrentStatus`. Flagging now so
developer can add them in the same pass as the queue/error work rather than
retrofitting the table model later.

## 1. Main window

### 1.1 Layout

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  [+ 추가 ▾]  [❚❚ 일시정지]  [▶ 재개]  [🗑 삭제]              [⚙ 설정]        │  ← toolbar
├─────────────────────────────────────────────────────────────────────────────┤
│ 이름          │ 크기    │ 진행률        │ ↓속도    │ ↑속도    │피어│ 상태     │
├───────────────┼─────────┼───────────────┼──────────┼──────────┼────┼──────────┤
│ ubuntu-24.04… │ 5.8 GB  │ [███████ 62%] │ 3.2 MB/s │ 128 KB/s │ 14 │ 다운로드중│
│ debian-net…   │ 654 MB  │ [██████████]  │    -     │  45 KB/s │  6 │ 시딩중    │
│ archlinux-2…  │ 900 MB  │ [░░░░░░░  0%] │    -     │    -     │  0 │ 대기중(큐)│
│ old-project…  │ 1.2 GB  │ [███░░░  35%] │    -     │    -     │  0 │ 일시정지  │
├─────────────────────────────────────────────────────────────────────────────┤
│ 전체: ↓ 3.2 MB/s  ↑ 173 KB/s   활성 1/4   프록시: ● 연결됨 (kill switch on)   │  ← status bar
```

### 1.2 Toolbar actions

| 버튼 | 동작 | 활성화 조건 |
|---|---|---|
| **추가 ▾** | 드롭다운: "`.torrent` 파일 열기…", "Magnet 링크 추가…" 두 항목. 각각 §2 다이얼로그를 다른 초기 탭으로 연다. | 항상 활성 (MUST) |
| **일시정지** | 선택된 항목 중 `is_paused == False`인 것을 모두 일시정지 | 선택된 항목 중 하나 이상이 실행 중일 때만 활성. 선택 없으면 비활성 (MUST) |
| **재개** | 선택된 항목 중 `is_paused == True`인 것을 모두 재개 | 선택된 항목 중 하나 이상이 일시정지 상태일 때만 활성 (MUST) |
| **삭제** | §3 삭제 확인 다이얼로그를 연다 (선택 항목 전체 대상) | 선택 항목 1개 이상일 때 활성 (MUST) |
| **설정** | §4 설정 다이얼로그를 연다 | 항상 활성 (MUST) |

다중 선택 시 일시정지/재개 버튼은 "혼합 상태"에서도 활성 상태를 유지하며, 각 항목에
개별적으로 적용되는 동작만 수행한다 (이미 일시정지된 항목에 일시정지를 다시 적용하지 않음).

### 1.3 테이블 컬럼 정의 (QTableView)

| 컬럼 | 소스 필드 | 정렬 | 포맷 | 리사이즈 | 정렬(sort) 키 |
|---|---|---|---|---|---|
| 이름 | `name` | 좌 | 전체 이름. 컬럼 폭보다 길면 말줄임(…), 툴팁에 전체 이름 표시 (SHOULD) | 가변폭, 창 확장/축소 시 이 컬럼이 남는 공간을 흡수 (stretch) | 문자열 (대소문자 무시) |
| 크기 | `total_bytes` | 우 | 사람이 읽기 쉬운 단위: `< 1 KB`→"B", `< 1 MB`→"KB", `< 1 GB`→"MB", 이상 "GB" (소수점 1자리, 예: `5.8 GB`) | 고정폭(리사이즈 가능), 최소 70px | 바이트 수치 |
| 진행률 | `progress` | 중앙 | 프로그레스바 위에 `%` 텍스트 오버레이 (`62%`). 소수점 없이 반올림 | 고정폭 120px, 최소 80px | 0.0–1.0 수치 |
| ↓속도 | `download_rate` | 우 | `KB/s` / `MB/s`. `0`이면 "`-`"로 표시 (파싱 완료·시딩·일시정지 시) | 고정폭(리사이즈 가능) | 정수 수치 |
| ↑속도 | `upload_rate` | 우 | 위와 동일 규칙 | 고정폭(리사이즈 가능) | 정수 수치 |
| 피어 | `num_peers` | 중앙 | 정수. 0이면 "0" (하이픈 아님 — 연결 실패와 구분하기 위해 숫자 그대로 노출) | 고정폭, 최소 40px | 정수 |
| 상태 | `state` + `is_paused` (+ `error_message`, `queue_position` — OPEN-1) | 중앙 | §5 상태 매핑 텍스트. 색상/배지는 design-publisher 결정 (의미만: 진행=강조색, 대기/일시정지=중립색, 완료/시딩=성공색, 오류=위험색) | 고정폭(리사이즈 가능), 최소 90px | §5의 상태 우선순위 (오류 > 다운로드중 > 대기중 > 일시정지 > 시딩 > 완료), 동률 시 이름순 (SHOULD; MVP 최소 요건은 문자열 정렬로도 충분) |

- 모든 컬럼 헤더 클릭 시 오름차순/내림차순 토글 정렬, 헤더에 정렬 방향 화살표 표시 (MUST — SCOPE #3 "live list" 요건의 일부로 목록 가독성에 필수).
- 초기 정렬: 추가된 순서(삽입 순서) 오름차순.
- 컬럼 폭 드래그 리사이즈 가능 (MUST, QTableView 기본 제공). 컬럼 폭을 `QSettings`에 저장해 재실행 시 복원 (SHOULD — SCOPE 11개 기준에는 없으나 저비용 개선).
- 행 선택: 단일 클릭 = 단일 선택, `Ctrl`+클릭 = 토글 추가, `Shift`+클릭 = 범위 선택 (MUST, 다중 일시정지/재개/삭제에 필요).
- 목록이 비어 있을 때: 테이블 대신 안내 문구 "추가된 토렌트가 없습니다. `.torrent` 파일을 열거나 Magnet 링크를 추가하세요." + "추가" 버튼 바로가기 (SHOULD).

### 1.4 상태바 (하단)

| 항목 | 내용 | 갱신 주기 |
|---|---|---|
| 전체 다운로드 속도 | 모든 토렌트의 `download_rate` 합, 컬럼과 동일 단위 포맷 | 1초 (MUST — SCOPE #3) |
| 전체 업로드 속도 | 모든 토렌트의 `upload_rate` 합 | 1초 |
| 활성/전체 수 | "활성 `N`/전체 `M`" — 활성 = 상태가 다운로드중/시딩중인 것 (일시정지·대기·오류 제외) | 1초 |
| 프록시 상태 | 설정에서 SOCKS5가 구성되지 않은 경우: "프록시: 미설정". 구성되어 연결된 경우: "프록시: ● 연결됨". 프록시 연결 실패로 kill switch가 트래픽을 차단 중인 경우: "프록시: ⚠ 연결 끊김 — 트래픽 차단됨" (경고색) | 프록시 상태 변경 시 즉시, 그 외 1초 폴링 (MUST — SCOPE #8) |

### 1.5 우클릭 컨텍스트 메뉴 (테이블 행)

행(들)을 우클릭하면 선택이 해당 행으로 갱신된 뒤(이미 선택돼 있으면 유지) 아래 메뉴가 뜬다:

```
┌────────────────────────┐
│  일시정지               │  ← 선택 항목 중 실행중인 것이 있을 때만 활성 (MUST)
│  재개                   │  ← 선택 항목 중 일시정지된 것이 있을 때만 활성 (MUST)
├────────────────────────┤
│  삭제…                  │  ← §3 다이얼로그 오픈, 툴바 삭제와 동일 동작 (MUST)
├────────────────────────┤
│  저장 폴더 열기          │  ← 탐색기에서 save_path 오픈, 단일 선택일 때만 표시 (SHOULD)
│  정보 (info hash 복사)  │  ← info_hash를 클립보드에 복사, 단일 선택일 때만 표시 (SHOULD)
└────────────────────────┘
```

컨텍스트 메뉴의 삭제는 별도 흐름을 만들지 않고 툴바 삭제와 **동일한 §3 다이얼로그**를
연다 — 삭제 확인 경로를 하나로 유지해 "안전한 기본값"이 항상 적용되게 한다.

## 2. 토렌트 추가 다이얼로그

### 2.1 구조 — 탭 방식

두 입력 방식(파일 선택 vs 텍스트 붙여넣기)은 상호작용 형태가 달라 하나의 폼에 억지로
합치면 어느 쪽을 채워야 하는지 헷갈린다. **탭으로 분리**하고, 저장 경로·옵션은 탭
아래 공통 영역에 둔다 (탭을 바꿔도 유지됨).

```
┌───────────────────────────────────────────────────┐
│  토렌트 추가                                   [x] │
├───────────────────────────────────────────────────┤
│  [ .torrent 파일 ] [ Magnet 링크 ]   ← 탭          │
│ ┌─────────────────────────────────────────────┐   │
│ │  (탭1) 파일 경로: [______________] [찾아보기] │   │
│ │  (탭2) Magnet URI:                            │   │
│ │        [________________________________]   │   │
│ │        [ 클립보드에서 붙여넣기 ]              │   │
│ └─────────────────────────────────────────────┘   │
│                                                     │
│  저장 경로: [ D:\Downloads\pytorrent    ] [찾아보기]│
│  [ ] 일시정지 상태로 추가                          │
│                                                     │
│                              [ 취소 ]   [ 추가 ]   │
└───────────────────────────────────────────────────┘
```

### 2.2 탭 1 — `.torrent` 파일

- "찾아보기" 클릭 시 표준 Windows 파일 열기 다이얼로그, 필터 `*.torrent` (MUST — SCOPE #1).
- 파일 선택 즉시 다이얼로그가 파일을 파싱해 이름을 미리 보여준다 (파싱 실패 시 §6 참조).
- 파일 경로 텍스트 필드에 직접 경로를 타이핑/붙여넣기도 허용 (SHOULD).

### 2.3 탭 2 — Magnet 링크

- 멀티라인은 불필요, 단일 라인 입력 (긴 magnet URI도 한 줄로 스크롤).
- 입력값이 `magnet:?xt=urn:btih:` 로 시작하지 않으면 필드 아래 인라인 오류 표시:
  "유효한 magnet 링크가 아닙니다" (실시간, 포커스 아웃 시 검증) — "추가" 버튼은 비활성화 상태 유지 (MUST, SCOPE #2와 §6 엣지케이스 연계).
- "클립보드에서 붙여넣기" 버튼: 클립보드 내용이 magnet URI 패턴이면 필드에 채움, 아니면 토스트 "클립보드에 magnet 링크가 없습니다" (SHOULD).

### 2.4 공통 영역

- **저장 경로**: 기본값은 설정(§4)의 "기본 저장 경로". 사용자가 이 필드를 바꾸면 그
  값이 **현재 앱 세션 동안** 다음 "추가" 호출의 기본값이 된다 (앱 재시작 시 설정의
  기본값으로 되돌아감). 설정 화면의 기본값 자체를 바꾸려면 §4에서 변경해야 한다.
  (**OPEN-2**: "세션 동안만 기억" vs "매번 설정에 영구 반영" 중 product-lead 확인 필요.
  기본값은 전자로 제안 — 다른 폴더에 한 번만 받는 경우가 흔해 설정이 오염되지 않음.)
- **일시정지 상태로 추가** 체크박스: 체크 시 추가된 토렌트는 즉시 `is_paused=True`로
  시작 — 메타데이터만 받아두고 다운로드는 나중에 시작하고 싶은 경우 (SHOULD, SCOPE
  명시 기준은 아니지만 프롬프트에서 명시적으로 요청됨).
- **추가** 버튼: 탭1에서는 파일 경로가 유효한 기존 파일일 때, 탭2에서는 magnet URI가
  유효한 패턴일 때만 활성화. 저장 경로가 비어 있으면 항상 비활성 + 안내 텍스트
  "저장 경로를 선택하세요".
- **취소**: 입력값 버리고 닫음. 확인 없이 즉시 닫힘 (파괴적 동작 아님).

### 2.5 Magnet 메타데이터 수신 전 UI 표현

"추가" 클릭 즉시 다이얼로그는 닫히고, 메인 테이블에 **행이 즉시 생성**된다 (사용자가
"먹통"이라고 느끼지 않도록). 메타데이터 도착 전 상태:

| 컬럼 | 표시 |
|---|---|
| 이름 | `(메타데이터 수신 중…)` (engine의 fallback과 동일 문구) |
| 크기 | `-` |
| 진행률 | 인디터미닛(불확정) 바 애니메이션, % 텍스트 없음 |
| ↓/↑속도 | `-` |
| 피어 | DHT/트래커에서 발견한 피어 수 (0 이상 가능 — 메타데이터 없이도 피어 탐색은 진행됨) |
| 상태 | `메타데이터 수신 중` |

메타데이터 수신 완료 시 다음 1초 폴링에서 자동으로 정상 행으로 전환된다 (별도 알림 불필요, MUST — SCOPE #2).

## 3. 삭제 확인 다이얼로그

```
┌───────────────────────────────────────────────────┐
│  토렌트 삭제                                   [x] │
├───────────────────────────────────────────────────┤
│  선택한 2개 항목을 삭제합니다:                     │
│    • ubuntu-24.04-desktop-amd64.iso                │
│    • debian-netinst.iso                            │
│                                                     │
│  ( ● ) 목록에서만 제거                             │
│        내려받은 파일은 디스크에 그대로 남습니다.   │
│                                                     │
│  ( ○ ) 데이터까지 삭제                             │
│        ⚠ 디스크의 파일도 함께 삭제됩니다.          │
│           이 작업은 되돌릴 수 없습니다.            │
│                                                     │
│                              [ 취소 ]   [ 삭제 ]   │
└───────────────────────────────────────────────────┘
```

- 기본 선택된 라디오: **목록에서만 제거** (안전한 기본값, MUST — 실수로 데이터를 잃지
  않도록).
- "데이터까지 삭제"를 선택하면 ⚠ 경고 문구가 더 강조된 스타일로 표시되고, "삭제"
  버튼 라벨이 "데이터까지 삭제" 선택 시 "영구 삭제"로 바뀐다 (SHOULD — 사용자가
  버튼을 누르는 순간 자신이 무엇을 확정하는지 재확인).
- 다중 선택 삭제 시 대상 목록을 스크롤 가능한 리스트로 표시 (5개 초과 시 "외 N개"로 축약, SHOULD).
- 삭제 진행 중 실패(예: 파일이 다른 프로세스에서 사용 중이라 삭제 불가) 시, 다이얼로그를
  닫지 않고 해당 항목에 인라인 오류 표시 + "재시도"/"목록에서만 제거로 전환" 선택지
  제공 (MUST 수준은 아니나 §6에서 상세; 최소 요건은 실패를 조용히 무시하지 않고
  사용자에게 알리는 것).

## 4. 설정 다이얼로그

탭 또는 섹션 구분 없이 스크롤 가능한 단일 폼으로 구성한다 (MVP 설정 항목 수가 적어
탭이 과함). 저장은 "저장"을 눌러야 반영되며, 프록시/포트 변경은 세션 재시작 없이
즉시 적용 (libtorrent `apply_settings`가 런타임 적용 가능).

```
┌───────────────────────────────────────────────────┐
│  설정                                          [x] │
├───────────────────────────────────────────────────┤
│  일반                                              │
│  기본 저장 경로: [ D:\Downloads\pytorrent ] [찾아보기]│
│  리스닝 포트:    [ 6881 ]                          │
│                                                     │
│  다운로드                                          │
│  [ ] 순차 단독 큐 (한 번에 하나씩 다운로드)        │
│                                                     │
│  프라이버시 — SOCKS5 프록시                        │
│  [ ] 프록시 사용                                   │
│      호스트: [______________]  포트: [_____]       │
│      [x] Kill switch (프록시 끊기면 직접 연결 차단)│
│      상태: ● 연결됨 / ○ 미설정 / ⚠ 연결 실패        │
│                                                     │
│  완료 후 동작                                      │
│  ( ● ) 없음   ( ○ ) 앱 종료   ( ○ ) 시스템 종료    │
│                                                     │
│                              [ 취소 ]   [ 저장 ]   │
└───────────────────────────────────────────────────┘
```

| 항목 | 세부 규칙 |
|---|---|
| 기본 저장 경로 | 최초 실행 기본값: `%USERPROFILE%\Downloads\pytorrent-desktop` (없으면 생성). 존재하지 않거나 쓰기 권한 없는 경로 지정 시 "저장" 클릭 시점에 검증, 실패하면 인라인 오류 "이 경로에 쓸 수 없습니다" + 저장 차단 (MUST 수준 검증 — §6 권한 없음 케이스와 연결). |
| 리스닝 포트 | 1024–65535 범위 검증, 범위 밖 입력 시 인라인 오류. 기본값 6881 (engine 기본값과 동일) |
| 순차 단독 큐 | 체크 시 `set_sequential_queue(True)` 즉시 호출 — MUST, SCOPE #6. 켜져 있을 때 이미 여러 개가 동시에 다운로드 중이었다면, 저장 직후 1초 폴링에서 큐 규칙이 반영되어 상위 큐 순번 하나만 남고 나머지는 "대기중(큐)"로 전환됨을 사용자가 목록에서 바로 확인 가능해야 함. |
| 프록시 사용 + 호스트/포트 | "프록시 사용" 체크 해제 시 호스트/포트/kill switch 필드 비활성화(회색). 체크 시 호스트/포트 필수 입력 — 비어 있으면 "저장" 비활성 + 인라인 오류. 저장 시 `configure_privacy()` 호출 (MUST, SCOPE #8) |
| Kill switch | 기본값 **On** (안전 기본값). 끄면 경고 문구 노출: "프록시 연결이 끊기면 실제 IP로 직접 연결될 수 있습니다." (SHOULD, 사용자가 명시적으로 리스크를 인지하게) |
| 프록시 상태 표시 | 설정 화면에서도 §1.4 상태바와 동일한 3-상태(미설정/연결됨/연결실패) 표시 (SHOULD) |
| 완료 후 동작 | 라디오 3택 1, 기본값 "없음". "앱 종료" 또는 "시스템 종료" 선택 시 아래 안내 문구 노출: "모든 다운로드가 끝나면 자동으로 실행됩니다. 실행 전 취소할 수 있는 확인 창이 표시됩니다." (§5.6 참조) (MUST, SCOPE #9) |
| 취소 | 변경사항 버리고 닫음 |
| 저장 | 유효성 검증 통과 시에만 적용 및 다이얼로그 닫힘 |

**OPEN-3 (to product-lead):** "시스템 종료" 선택은 다른 애플리케이션·저장되지 않은
작업에 영향을 줄 수 있는 파괴적 동작이다. 이 스펙은 §5.6에 "실행 전 취소 가능한
카운트다운 확인 창"을 기본 설계로 제안한다(사용자가 자리를 비운 사이 완료되어도
즉시 종료되지 않고 일정 시간 유예). SCOPE에는 이 확인 절차가 명시돼 있지 않으므로
product-lead의 확인을 받아야 한다 — 확인 창 없이 즉시 종료하는 대안도 가능하지만
데이터 손실 리스크가 있어 권장하지 않는다.

## 5. 상태 전이

### 5.1 상태 목록과 목록 표시

| 내부 상태 (engine `state` / 파생) | 목록 표시 라벨 | 진행률 바 | 속도 표시 | 피어 |
|---|---|---|---|---|
| `downloading_metadata` | 메타데이터 수신 중 | 인디터미닛 | `-` / `-` | 발견된 피어 수 |
| `checking_files` / `checking_resume_data` | 파일 확인 중 | 확인 진행률(%) | `-` / `-` | 이전 세션 값 유지 또는 0 |
| `downloading` (큐 대기 아님) | 다운로드 중 | 실측 % | 실측 값 | 실측 값 |
| 큐 대기 — `queue_position > 0`이고 아직 활성화되지 않음 (**OPEN-1** 필드 필요) | 대기 중 (큐) | 마지막 진행률 유지 | `-` / `-` | 0 |
| `finished` | 완료 | 100% | `-` / 실측 업로드(시딩 안 해도 잔여 전송 가능) | 실측 값 |
| `seeding` | 시딩 중 | 100% | `-` / 실측 값 | 실측 값 |
| `is_paused == True` (다른 모든 상태를 덮어씀) | 일시정지 | 마지막 진행률 유지(회색조) | `-` / `-` | 0 또는 마지막 값 유지 |
| 오류 (**OPEN-1** `error_message` 필요) | 오류 | 마지막 진행률 유지(빨간 테두리) | `-` / `-` | 0 |

`is_paused`는 다른 모든 상태 표시를 덮어쓴다 — 예: 시딩 중 토렌트를 일시정지하면
"시딩 중"이 아니라 "일시정지"로 표시된다. 단, 오류는 일시정지보다 우선한다 (오류가
난 토렌트는 일시정지 여부와 무관하게 "오류"로 표시 — 사용자가 문제를 놓치지 않도록).

**상태 우선순위 (표시 결정 순서):** 오류 > 일시정지 > (메타데이터수신중 > 파일확인중
> 대기중(큐) > 다운로드중 > 완료 > 시딩중)

### 5.2 정상 흐름 (`.torrent` 파일, SCOPE #1)

파일 선택 → 저장 경로 확인 → `파일 확인 중` (기존 데이터가 있을 때만, 신규는 생략
가능) → `다운로드 중` → `완료` → (시더가 남아 있고 자동 시딩이 켜져 있으면) `시딩 중`

### 5.3 정상 흐름 (magnet, SCOPE #2)

Magnet 붙여넣기 → `메타데이터 수신 중` (§2.5 임시 행) → 메타데이터 도착 시 이름/크기
확정 → `다운로드 중` → `완료` → `시딩 중`

### 5.4 일시정지/재개 (SCOPE #4)

임의 상태에서 "일시정지" 클릭 → 즉시(다음 1초 폴링 내) `일시정지`로 표시, 속도 0으로
표시. "재개" 클릭 → 일시정지 이전 상태로 복귀(다운로드중 이어서 재개, 또는 완료된
토렌트면 시딩중으로).

### 5.5 큐 (SCOPE #6)

설정에서 "순차 단독 큐" On, 토렌트 3개가 다운로드 대기 중이라고 하자:
1. 큐 순번 0(가장 먼저 추가/우선순위 높은 것)만 `다운로드 중`, 나머지 2개는 `대기 중(큐)`.
2. 순번 0이 `완료`되면, 다음 1초 폴링 내에 순번 1이 자동으로 `다운로드 중`으로 전환.
3. 사용자가 대기 중인 항목을 수동으로 "재개"하거나 우선순위를 올리는 조작은 MVP
   범위 밖 (post-MVP: 큐 순서 변경 UI). MVP에서 순서는 항상 추가된 순서를 따른다.

### 5.6 완료 후 동작 실행 (SCOPE #9)

설정에서 "앱 종료" 또는 "시스템 종료"가 선택된 상태에서, **모든** 토렌트가 `완료`
또는 `시딩 중` 상태가 된 순간(하나라도 `다운로드 중`/`메타데이터 수신 중`/`대기 중`이면
트리거되지 않음):

```
┌───────────────────────────────────────────┐
│  모든 다운로드 완료                        │
├───────────────────────────────────────────┤
│  30초 후 시스템을 종료합니다.              │
│                                             │
│              [ 지금 취소 ]                 │
└───────────────────────────────────────────┘
```

- 카운트다운 30초 (OPEN-3에서 product-lead 확인 필요한 기본값), "지금 취소" 클릭 시
  다이얼로그 닫히고 완료 후 동작은 이번 세션에서 더 이상 트리거되지 않음(단, 새
  토렌트를 추가해 다시 미완료 상태로 돌아갔다가 재차 모두 완료되면 다시 트리거).
- 카운트다운 중 새 토렌트가 추가되어 "완료"가 아닌 상태가 생기면 자동으로 다이얼로그
  취소 (사용자가 취소를 누르지 않아도 안전).
- 카운트다운 종료 시 앱 종료(`QApplication.quit()`) 또는 시스템 종료(Windows
  `shutdown /s /t 0` 또는 API) 실행.

## 7. 수용 테스트 시나리오 (SCOPE.md MVP 기준 1–11 → 실행 가능한 QA 스텝)

각 시나리오는 "사용자 조작 → 기대 결과"로 qa-tester가 그대로 실행할 수 있게 작성.

**AC-1. `.torrent` 파일 열기**
1. 툴바 "추가 ▾" → ".torrent 파일 열기…" 클릭 → 파일 다이얼로그가 뜬다.
2. 합법적인 `.torrent` 파일(예: Ubuntu ISO torrent) 선택 → §2 추가 다이얼로그가
   해당 탭 활성 상태로 열리고 저장 경로가 기본값으로 채워져 있다.
3. 저장 경로 확인 후 "추가" 클릭 → 다이얼로그가 닫히고 메인 테이블에 새 행이 즉시
   나타난다.
4. **기대 결과**: 몇 초 내 상태가 `다운로드 중`으로 바뀌고 ↓속도가 0보다 커진다.

**AC-2. Magnet 링크 추가**
1. "추가 ▾" → "Magnet 링크 추가…" → 유효한 magnet URI 붙여넣기 → "추가" 클릭.
2. **기대 결과**: 테이블에 `메타데이터 수신 중` 행이 즉시 나타난다.
3. 몇 초~몇 분 대기.
4. **기대 결과**: 이름/크기가 실제 값으로 갱신되고 상태가 `다운로드 중`으로 전환,
   ↓속도가 0보다 커진다.

**AC-3. 실시간 목록 갱신**
1. 다운로드 중인 토렌트가 1개 이상 있는 상태에서 테이블을 10초간 관찰.
2. **기대 결과**: 이름/크기/진행률/↓속도/↑속도/피어/상태 값이 약 1초 간격으로
   변하는 것을 육안으로 확인 (진행률 %가 증가, 속도 수치가 갱신됨).

**AC-4. 일시정지/재개**
1. 다운로드 중인 토렌트를 선택 → 툴바 "일시정지" 클릭.
2. **기대 결과**: 상태가 `일시정지`로 바뀌고 ↓/↑속도가 `-`가 된다.
3. 같은 항목 선택 → "재개" 클릭.
4. **기대 결과**: 상태가 `다운로드 중`(또는 완료돼 있었다면 `시딩 중`)으로 돌아가고
   속도가 다시 0보다 커진다.

**AC-5. 삭제 (목록만 vs 데이터 포함)**
1. 토렌트 선택 → "삭제" 클릭 → §3 다이얼로그에서 "목록에서만 제거" 선택(기본값
   유지) → "삭제" 클릭.
2. **기대 결과**: 테이블에서 행이 사라지고, 탐색기에서 저장 경로를 확인하면 파일이
   그대로 남아 있다.
3. 다른 토렌트를 추가 → 선택 → "삭제" → 이번엔 "데이터까지 삭제" 선택 → "삭제"(또는
   "영구 삭제") 클릭.
4. **기대 결과**: 테이블에서 행이 사라지고, 탐색기에서 저장 경로를 확인하면 해당
   파일/폴더도 삭제되어 있다.

**AC-6. 순차 단독 큐**
1. 설정 → "순차 단독 큐" 체크 → 저장.
2. 서로 다른 토렌트 3개를 연달아 추가.
3. **기대 결과**: 테이블에서 동시에 `다운로드 중`인 항목이 정확히 1개뿐이고, 나머지
   2개는 `대기 중(큐)`로 표시된다.
4. 다운로드 중인 항목이 `완료`(또는 `시딩 중`)로 바뀔 때까지 대기.
5. **기대 결과**: 다음 1초 폴링 내에 대기 중이던 항목 중 하나가 자동으로
   `다운로드 중`으로 전환된다.

**AC-7. 세션 복원**
1. 진행 중(완료 전)인 토렌트가 있는 상태에서 앱을 정상 종료.
2. 앱을 다시 실행.
3. **기대 결과**: 종료 전 목록에 있던 토렌트가 그대로 나타나고, 진행률이 종료 시점
   근처에서 이어서 증가한다(처음부터 다시 받지 않는다).

**AC-8. SOCKS5 프록시 + kill switch**
1. 설정 → "프록시 사용" 체크, 유효한 SOCKS5 host/port 입력, kill switch 체크
   유지 → 저장.
2. **기대 결과**: 상태바 프록시 표시가 "● 연결됨"이 된다.
3. 프록시 서버를 중단시키거나 잘못된 host/port로 바꿔 저장.
4. **기대 결과**: 상태바가 "⚠ 연결 실패 — 트래픽 차단됨"으로 바뀌고, 모든 토렌트의
   피어 수가 0으로 떨어진다(네트워크 모니터링 툴로 실제 트래픽이 프록시 없이 직접
   나가지 않는지 확인 — kill switch 핵심 검증).

**AC-9. 완료 후 동작(옵트인)**
1. 설정 → "완료 후 동작"에서 "앱 종료" 선택 → 저장.
2. 크기가 작은 토렌트 1개만 추가해 완료까지 대기.
3. **기대 결과**: 완료 즉시(또는 완료 조건 충족 시) §5.6의 카운트다운 다이얼로그가
   뜬다.
4. 아무 조작 없이 카운트다운 종료까지 대기.
5. **기대 결과**: 앱이 자동으로 종료된다. (시스템 종료 옵션은 별도로 반복 검증하되
   실제 종료는 테스트 환경에 영향을 주므로 카운트다운 다이얼로그 노출까지만
   확인하고 "취소"로 중단해도 무방.)

**AC-10. 독립 실행형 `.exe`**
1. Python이 설치되지 않았거나 별도 가상환경이 없는 클린 Windows 머신(또는 VM)에
   `pytorrent-desktop.exe`(PyInstaller 빌드)를 복사.
2. 더블클릭 실행.
3. **기대 결과**: 별도 설치/설정 없이 메인 윈도우가 뜨고 AC-1~AC-9가 모두 동일하게
   동작한다.

**AC-11. 실제 합법 토렌트 100% 완주**
1. Ubuntu ISO 등 합법적인 실제 토렌트를 추가.
2. 완료(진행률 100%, 상태 `완료` 또는 `시딩 중`)까지 대기.
3. **기대 결과**: 다운로드된 파일의 체크섬(예: SHA256)이 공식 배포처의 체크섬과
   일치한다 — 데이터 무결성까지 검증.

## 8. 와이어프레임 (텍스트/ASCII)

이미 §1–§5 각 섹션에 화면별 ASCII 레이아웃을 포함했다. 전체를 한눈에 보기 위한
요약 인덱스:

- 메인 윈도우: §1.1
- 토렌트 추가 다이얼로그: §2.1
- 삭제 확인 다이얼로그: §3
- 설정 다이얼로그: §4
- 완료 후 동작 카운트다운 다이얼로그: §5.6

design-publisher는 위 레이아웃의 요소 배치·간격·계층 구조를 기준으로 실제 시각
디자인(색상, 타이포그래피, 아이콘, spacing 토큰)을 정하면 된다. 색상은 이 스펙에서
의도적으로 지정하지 않았다(§1.3, §5.1의 "의미만" 표기 참조) — 시맨틱 의도만 전달.
